Obsidian
Telemetry

A localized mesh network interface for coordinated drone movement in GPS-denied environments. Brutalist data visualization for high-latency systems.

01 //

System Architecture

The core requirement for Obsidian Telemetry was a zero-dependency architecture capable of running on embedded hardware. We utilized a custom FastAPI implementation stripped of middleware to ensure microsecond response times. The frontend renders purely based on websocket streams, bypassing traditional REST polling.

class TelemetryNode:

        self.id = node_id

    # Optimized for low-bandwidth environments
