MARL_FLOW_TRAINER

Decentralized Multi-Agent Reinforcement Learning Environment.

:: STACK: VUE_PYTHON | :: INFRA: DOCKER | :: STATUS: LOCAL_ONLY
// FIG.01: AGENT_TOPOLOGY

01 // CONTEXT

PAIN_POINT: THE_FLUENCY_GAP

Despite streaks in apps like Duolingo, users fail to articulate complex ideas when faced with open-ended conversation. The rigid curriculum structure creates a dependency on multiple-choice prompts, atrophying the brain's generative language centers.

MARL_FLOW_TRAINER abandons the linear curriculum. Instead, it deploys a swarm of LLM-driven agents that simulate a dynamic social environment. The user is not a student but an actor in a scenario, forced to negotiate meaning in real-time. The system prioritizes "communicative success" over grammatical perfection, mirroring natural language acquisition in immersion environments.

02 // PROCESS

01.

Context Generation

System procedurally generates a scenario (e.g., "Negotiating a refund at a cyberpunk bodega") and assigns hidden motives to NPC agents.

02.

Input Analysis

User audio is transcribed via Whisper. Semantic intent is parsed, not just syntax. The system evaluates if the goal was achieved.

03.

Feedback Loop

Post-interaction, the "Mentor Agent" breaks down the conversation, highlighting missed opportunities for nuance rather than just error correction.

03 // TOOLKIT

[ Vue 3 ] [ TailwindCSS ] [ Flask ] [ Python 3.10 ] [ Socket.IO ] [ Ollama ] [ OpenAI Whisper ] [ Raspberry Pi 4 ] [ SQLite ] [ ChromaDB ]

04 // META_DESIGN

The architecture of this project was iteratively refined using Gemini 1.5 Pro. By feeding the model architectural constraints (low latency, local-only privacy), we arrived at a split-brain topology where the "Reptilian" fast-response layer handles audio I/O on the edge, while the "Neocortex" slow-thinking layer (the LLM swarm) runs asynchronously, preventing UI blocking during inference spikes.
